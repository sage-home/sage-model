## Development Workflow Guidance

### **Starting a New Feature**
1. Read relevant architecture docs for your domain
2. Check `sage_unit_tests.md` for existing test patterns
3. Use `templates/` for consistent code structure
4. Follow `logging_guidelines.md` for debugging

### **Debugging Issues**
1. `evolution_diagnostics.md` for performance analysis
2. `logging_guidelines.md` for effective logging
3. `test_validation_framework.md` for validation approaches
4. Domain-specific docs for architecture understanding

### **Modifying Core Systems**
1. **REQUIRED**: `core_physics_separation.md` - understand separation principles
2. `property_dispatcher_system.md` if touching properties
3. `config_system.md` if modifying configuration
4. `pipeline_phases.md` if changing execution flow

### **Adding I/O Support**
1. `io_interface_migration_guide.md` - migration strategy and patterns
2. `io_hdf5_output_guide.md` - output API design
3. `property_dispatcher_system.md` - serialization system
